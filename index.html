Evolution of Javascript
Browser Wars Era
Once upon a time...
Microsoft and Netscape fought over the Browser Market Share
Mocha was born in March/April 1995, hacked up in about 10 days for Netscape Navigator
which was then renamed to LiveScript in September 1995, which then settled on the name JavaScript in December 1995
The name implied a connection to Java, which is what Netscape intended, even though the language had nothing to do with Java. The renaming also coincided with Netscape bundling support for Java in it's browser, adding to the confusion. Submitted to ECMA - 1996-97.
Microsoft launched VBScript early 1996 (for Windows Scripting), bundling it with IE shortly. It also adopted JavaScript as JScript.
VBScript was considered an equal contender to Javascript around then, and gained a fair bit of traction as well. Partly owing to it's use for scripting Office apps. And also because VB was supposedly simpler. ASP used VBScript. But it ran only on IE (which didn't matter much owing to IE's major market share).
Then came Firefox (2002), making a serious dent in the browser market.
JavaScript started off it's life backing simple tasks: UI transitions, automating simple calculations...
...and toy scripts like this http://rainbow.arch.scriptmania.com/scripts/mouse_clock3.html
